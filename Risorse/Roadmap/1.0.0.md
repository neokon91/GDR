# Roadmap 1.0.0

Obiettivo: trasformare il vault in un prodotto professionale, non in una raccolta di dashboard. La prossima conversazione deve partire da qui e non riaprire analisi generiche.

## Direzione Vincolante

Il prodotto deve funzionare come:

1. **Obsidian TTRPG runtime**: preparo, gioco, chiudo la sessione.
2. **World Anvil locale**: creo e consulto un codex mondo leggibile, player-safe e collegato.

Tutto cio che non aiuta questi due risultati va nascosto, collassato o rimandato.

## Non Fare

- Non aggiungere nuove dashboard generiche.
- Non aggiungere nuova documentazione prodotto.
- Non creare nuovi hub.
- Non moltiplicare indici.
- Non aggiungere template se non riducono passaggi reali.
- Non mettere strumenti avanzati nel flusso principale.

## Priorita 1 - Flusso Sessione Chiuso

Scopo: un DM deve creare o aprire una sessione, compilarla, giocarla e chiuderla senza girare tra dieci note.

- [x] Ridurre [[Inizia Qui]] al flusso Prepara -> Gioca -> Aggiorna il mondo.
- [x] Rendere [[z.modelli/dm/Sessione]] compilabile subito con 5 campi: obiettivo, prima scena, scelta, pressione, materiale.
- [x] Creare output “scaletta giocabile” dentro la nota sessione usando quei 5 campi.
- [x] Rendere chiaro in ogni vista se esiste una sessione attiva e quale e.
- [x] Aggiungere azione/istruzione chiara per impostare una sola sessione `attiva: true`.
- [x] Ridurre [[Durante il Gioco]] a cockpit: scena corrente, appunti live, decisioni, clock, handout, PNG presenti.
- [x] Spostare tabelle lunghe e archivi fuori dal primo schermo di [[Durante il Gioco]].
- [x] Rendere “Fine sessione” un passaggio visibile da [[Durante il Gioco]].

## Priorita 4 - Atlante Mappe

Scopo: non “abbiamo plugin mappe”, ma un atlante usabile.

- [ ] Definire una mappa principale demo del mondo.
- [ ] Definire una mappa player-safe demo.
- [ ] Standardizzare campi mappa: uso, pubblico, mondo, luoghi, layer, pin, versione giocatori.
- [ ] Controllare luoghi senza mappa e mappe senza luoghi.
- [ ] Collegare rotte, territori e fazioni alla vista atlante.

## Priorita 5 - Player Portal

Scopo: una vista giocatori realmente mostrabile senza esporre segreti.

- [ ] Rendere [[Vista Giocatori]] una home player-facing pulita, non una dashboard tecnica.
- [ ] Mostrare recap pubblici ordinati.
- [ ] Mostrare missioni attive player-safe.
- [ ] Mostrare PNG noti e luoghi noti come card.
- [ ] Mostrare handout consegnati.
- [ ] Mostrare mappa pubblica centrale.
- [ ] Rafforzare controllo anti-segreti.
- [ ] Valutare export/Publish-ready solo dopo che la vista interna e solida.

## Priorita 6 - Party Control

- [ ] HP editabili chiaramente.

## Priorita 7 - UX E Visual Polish

- [ ] Ridurre tabelle nel primo schermo delle pagine operative.

## Priorita 8 - Demo Prodotto

Scopo: la demo deve vendere il vault in 2 minuti.

- [ ] Sessione demo pronta con 5 campi completi.
- [ ] Cockpit demo utilizzabile.
- [ ] Post-sessione demo con conseguenze propagate.
- [ ] Codex demo mondo completo.
- [ ] Player portal demo popolato.
- [ ] Mappa demo forte.
- [ ] Party demo completo.

## QA Obbligatoria

Prima di qualunque release:

- [ ] Nuovo utente capisce il prossimo click in 30 secondi.
- [ ] Una sessione minima si rende pronta in una schermata.
- [ ] Da home a tavolo servono massimo 2 click.
- [ ] Durante il gioco non serve aprire archivi per appunti, decisioni, clock, handout.
- [ ] Post-sessione produce conseguenze e prossima sessione.
- [ ] Player portal non espone segreti.
- [ ] `npm run check` passa.
- [ ] `npm run release:clean` passa.

## Stato Attuale

Base tecnica presente, prodotto incompleto. I prossimi interventi devono partire da **Priorita 1** e procedere in ordine. Non saltare a packaging, screenshot, marketing o nuove feature prima di avere un flusso sessione chiuso.
