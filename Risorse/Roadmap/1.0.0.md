# Roadmap 1.0.0

Obiettivo: preparare una release usabile senza frizione. La priorita non e aggiungere dashboard, ma rendere il flusso **Prepara -> Gioca -> Aggiorna il mondo** immediato, concreto e verificabile.

## Priorita

- [ ] QA operativa del percorso: [[Inizia Qui]] -> [[Risorse/Preparazione Sessione]] -> [[Durante il Gioco]] -> [[Cosa Succede Fuori Scena]].
- [ ] Ridurre o collassare ogni gruppo di pulsanti che non serve al passo corrente.
- [ ] Verificare che un nuovo DM capisca in 30 secondi quale click fare.
- [ ] Verificare che [[Risorse/Preparazione Sessione]] dica in una schermata cosa manca alla sessione.
- [ ] Trasformare dashboard consultive in strumenti con output o spostarle tra strumenti avanzati.
- [ ] QA visuale solo dopo UX operativa: [[Inizia Qui]], [[Risorse/Preparazione Sessione]], [[Durante il Gioco]], [[Cosa Succede Fuori Scena]], [[Vista Giocatori]].
- [ ] Screenshot o GIF per il flusso base, non per ogni dashboard: Prepara -> Gioca -> Aggiorna il mondo.
- [x] Ridurre `Inizia Qui` al flusso operativo Prepara -> Gioca -> Aggiorna il mondo.
- [x] Rendere [[Risorse/Preparazione Sessione]] una pagina con output concreto in cinque blocchi.
- [ ] Verificare che `npm run release:clean` includa tutte le nuove pagine utente e non includa roadmap interne.
- [x] Aggiornare messaggio pubblico, README release e note di installazione con focus living world, portale giocatori, party control e quality report.
- [ ] Preparare GitHub Release con ZIP pulito, changelog e limiti noti.
- [ ] Valutare export area giocatori come obiettivo post-1.0.

## Criterio Di Qualita

La release 1.0 e pronta quando un nuovo DM puo aprire il vault, cliccare **Prepara**, completare i cinque blocchi minimi, cliccare **Gioca**, poi cliccare **Aggiorna il mondo** senza leggere documentazione tecnica o aprire cinque dashboard diverse.

## Blocco Nuove Feature

Fino a quando il flusso base non supera il QA operativo, non aggiungere nuove dashboard, nuovi hub o nuovi template generici. Ogni modifica deve seguire [docs/UX_OPERATIVA.md](../../docs/UX_OPERATIVA.md).
