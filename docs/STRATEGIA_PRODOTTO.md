# Strategia Prodotto

## Executive Summary

Vault GDR e un vault Obsidian professionale per DM e worldbuilder italiani che vogliono preparare campagne, costruire ambientazioni profonde e giocare senza dover capire plugin, template o automazioni.

Il prodotto deve essere percepito come una app locale in Obsidian: dashboard, pulsanti, viste operative, controlli e release pulita. Non come un vault personale copiato da qualcuno.

## Positioning

**Categoria:** Obsidian TTRPG Operating System per DM non tecnici.

**Promessa:** apri il vault, scegli cosa vuoi fare, prepara o gioca usando dashboard guidate.

**Differenziazione:**

- esperienza in italiano;
- onboarding non tecnico;
- flusso completo da mondo a campagna a sessione;
- worldbuilding sistemico: tassonomia, relazioni, canone, geopolitica e mondo vivo;
- vista giocatori dedicata;
- release pulita;
- contenuti Markdown posseduti dall'utente.

| Target | Bisogno |

## Competitive Analysis

| Categoria competitor | Forza | Gap sfruttabile |

## SWOT

### Punti Forti

- Dashboard e flussi gia operativi.
- Forte differenziazione italiana.
- Worldbuilding profondo, persistente e collegato al tavolo.
- Motore Mondo Vivo e Geopolitical Dashboard gia presenti come differenziazione forte.
- Release pulita e controlli automatici.
- Vista giocatori gia presente.

### Punti Deboli

- Installazione ancora da rendere consumer-grade.
- Mancano screenshot e video.
- Dipendenza da molti strumenti Obsidian.
- Community governance ancora minima.

### Opportunita

- Pubblicazione su GitHub Releases, Vault Hub, itch.io e community Obsidian/TTRPG.
- Moduli premium futuri: campagne, ambientazioni, template avanzati.
- Pacchetti per generi diversi: fantasy classico, sandbox, investigativo, OSR.

### Rischi

- Plugin che cambiano comportamento.
- Utenti non tecnici bloccati al primo avvio.
- Percezione di complessita se il README non e chiaro.

## Gaps Principali

1. Release ZIP ufficiale e visibile.
2. Screenshot/GIF delle dashboard.
3. Screenshot/GIF di Motore Mondo Vivo e Geopolitical Dashboard.
4. README piu commerciale.
5. Export vista giocatori.
6. Controllo anti-segreti pubblici.
7. Demo video breve.
8. Branding piu memorabile di "Vault GDR".
9. Strategia core open source + moduli premium.

## Roadmap Strategica

### Breve Termine

- Release ZIP su GitHub.
- Screenshot principali.
- Demo visuale della catena evento -> fazione -> relazione -> territorio -> prossima sessione.
- README prodotto.
- CONTRIBUTING e issue template.

### Medio Termine

- Export area giocatori.
- Modalita preparazione rapida.
- Controlli narrativi avanzati.
- Demo video.
- Pubblicazione su canali community.

### Monetizzazione Futura

- Pack premium di campagne e ambientazioni.

- Supporto setup e personalizzazione.
- Patreon/Ko-fi per release anticipate e contenuti aggiuntivi.
