# z.bacheche

Bacheche Kanban operative del vault.

| Board | Uso |
| --- | --- |
| `Preparazione Sessioni.md` | Coda di lavoro prima della sessione. |
| `Post Sessione.md` | Consolidamento, canonizzazione e prossima apertura. |
| `Bacheca Creature.md` | Idee e stato delle creature. |
| `Manutenzione Vault.md` | Controlli tecnici e ricorrenze di manutenzione. |

Le bacheche sono Markdown con frontmatter `kanban-plugin: board`. Non usarle come archivio definitivo: le decisioni importanti devono finire in note canoniche dentro `Mondi/`, `Campagne/` o `Risorse/`.

Le checklist operative usano `#task` per essere compatibili con il plugin Tasks. Le checklist narrative e le checklist di documentazione restano senza tag, cosi non entrano per errore nel backlog del DM.

Usa priorita e scadenze solo per lavoro reale del DM o manutenzione. Non trasformare missioni, clock, conseguenze o eventi del mondo in task ricorrenti.
